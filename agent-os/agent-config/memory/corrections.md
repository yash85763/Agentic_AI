# Agent Corrections Log

This file is **human-editable**. Add corrections here when the agent makes systematic mistakes. The agent reads this file at the start of every run to avoid repeating mistakes.

---

## Format

```markdown
### [YYYY-MM-DD] [File/Dataset]: Issue Description
- **Problem**: What went wrong
- **Correct behavior**: What should happen instead
- **Example**: Optional before/after
```

---

## Guidance for Writing Corrections

- **Be specific**: include exact column names, file types, or business contexts
- **Be reusable**: the correction should help the agent generalize, not just patch one job
- **Be actionable**: describe what to do, not just what's wrong
- **Use evidence**: link to a specific job if relevant
